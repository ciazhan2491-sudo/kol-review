# 空間特工 · KOL / KOC 審核管理

把 KOL / KOC 合作流程從「經驗化」轉為「可追蹤、可交接」的單頁應用。

## 線上版（GitHub Pages）

部署後網址：`https://<你的帳號>.github.io/<repo 名稱>/`

> 開啟需連網（從 CDN 載入 React / Tailwind / 圖示）。資料目前存於瀏覽器 localStorage，**每台裝置各自獨立、不互通**。

## 功能

- **名單總表**：集中管理所有名單，含搜尋、狀態流轉、刪除
- **審核評分**：8 大維度各 1–5 分，滿分 40，自動判定合作等級並產生建議
- **合作追蹤**：寄送資料、發布任務、發布連結、補件提醒
- **成效紀錄**：依平台顯示對應欄位，自動計算 CTR / 互動率 / ROI
- **歷史紀錄**：每筆操作時間軸，方便交接回溯
- **角色權限**：管理員 / 專案負責人 / 協作執行，操作權限分級

## 兩種版本

| 檔案 | 說明 | 用途 |
|------|------|------|
| `index.html` / `kol-審核.html` | 單檔，免安裝，瀏覽器直接開 | 目前部署的線上版（GitHub Pages） |
| `src/` + Vite | 完整 React + Vite 專案 | 未來接雲端資料庫時的開發基礎 |

## 本機開發（Vite 版）

```bash
npm install
npm run dev
```

## 後續規劃（雲端）

目前資料存於 localStorage，無法多人共用。未來改接後端後即可團隊共用：
- Supabase / Firebase（前端直連，免自架伺服器）
- 或以 Google Sheet + Apps Script 作為輕量資料庫
